MCP WebSocket Hub — Test UI

Sample Flow: Zeroclaw ↔ Device

Follow these steps to test Zeroclaw interacting with a device via the hub.

1
Start Zeroclaw with MCP hub

Run zeroclaw serve (or gateway) with hub URL in config, then run zeroclaw register in another terminal to create a pairing code.

2
Confirm pairing

Copy the pairing code from section 1 below. In the Zeroclaw terminal running register, paste the code when prompted. Or call POST /api/v1/devices/register/confirm with the pairing code.

3
Device appears in Connected Devices

After pairing, the device shows in section 2. Click Call next to the device to load it into the MCP request form and fetch its tools.

4
Initialize MCP session

With device ID set, send initialize to handshake. Then send tools/list to see available tools.

5
Call a tool

Select a tool (e.g. file_read, memory_recall), set method to tools/call, adjust arguments if needed, then click Send.

1. Pending Pairing Codes

When zeroclaw register runs, pairing codes appear here and in the hub terminal.

2. Connected Devices

Call fills device ID in section 3. Revoke disconnects and removes credential; device must re-register.

3. Send MCP Request (as DatumBridge)

Sends JSON-RPC to device via hub proxy. Tools are auto-loaded when device ID is set.

For file tools: use relative paths (e.g. README.md). Absolute paths are blocked unless in [autonomy].allowed_roots.
